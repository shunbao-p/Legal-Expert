# 当前项目多模态处理问答（基于已实现代码）

## 1. 图片怎么处理？（OCR？还是 VLM？）
当前是 **OCR 路线**，不是 VLM。  
实现方式：使用 `Pillow + pytesseract` 提取图片文字，再进入文本切块与会话问答流程。

## 2. 表格怎么处理？（结构化？还是文本化？）
当前以 **文本化** 为主，不是结构化建模。  
实现方式：`Excel` 通过 `openpyxl` 按 `sheet` 和行读取，拼接为文本（例如 `[sheet]...`、`A | B | C`），再切块检索。

## 3. PDF 是怎么解析的？
使用 `pypdf.PdfReader` 按页执行 `extract_text()`，将全文拼接后切块，作为会话文档参与检索与生成。

## 4. 不同模态有没有“区别对待”？核心问题：是不是只是把所有东西变成文本？
有“区别对待”，主要发生在 **解析阶段**：
- PDF、Word、Excel、图片分别走不同解析器；
- 解析结果统一转为文本块 `Document`。

所以核心上，当前方案是：**多模态先转文本，再走统一的 RAG 检索与生成链路**。
